## 17:30 - API FastAPI criada e funcionando
### Estrutura
- `api/segmentor.py` — ClothesSegmentor class que encapsula GroundingDINO + SAM2 pipeline
- `api/server.py` — FastAPI server com `/segment` POST e `/health` GET
- `api/run_server.bat` — Script para abrir servidor em janela separada (porta 8001)
- `api/test_endpoint.py` — Test single image via HTTP
- `api/test_both.py` — Test both images via HTTP

### Problemas resolvidos
1. **numpy/h5py/tensorflow incompatibilidade**: numpy 2.4.6 incompatível com h5py 3.11.0 → downgrade para numpy<2 (1.26.4)
2. **Event loop bloqueado**: GroundingDINO+SAM2 são síncronos e pesados, bloqueiam o event loop do uvicorn → solução: `run_in_executor` com ThreadPoolExecutor
3. **Porta 8000 ocupada**: outro serviço rodando na porta 8000 → API configurada para porta 8001

### Testes HTTP — sucesso total
- `test_both.py` executado, ambas as imagens retornaram HTTP 200
- **1-TEST.jpg**: 27 objetos — hat(10), pants(3), dress(3), blouse(3), shirt(2), boots(2), shoes(1), sunglasses(1), handbag(1), skirt(1)
- **2-TEST.jpg**: 6 objetos — blouse(3), sunglasses(1), jacket(1), hat(1)
- Startup: ~36s para carregar modelos GroundingDINO + SAM2 tiny
